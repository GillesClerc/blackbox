# ADS7830

> _Synthèse générée par component-research. Datasheet PDF : [./ADS7830.pdf](./ADS7830.pdf)_

**Fabricant** : Texas Instruments
**Catégorie** : ADC 8 bits 8 canaux, interface I²C
**Référence officielle** : ADS7830IPWR (datasheet SBAS302C, rév. octobre 2012)
**Datasheet source** : https://www.ti.com/lit/ds/symlink/ads7830.pdf

## Vue d'ensemble

ADC SAR 8 bits, 8 entrées single-ended (ou 4 différentielles), 70 kSPS max,
piloté entièrement par I²C (standard / fast / high-speed). Référence interne
2,5 V débrayable **ou référence externe 0,05 V à VDD** — câblée sur VDD, la
mesure devient ratiométrique : idéale pour lire des potentiomètres alimentés
par le même rail. Micropuissance (~300 µW en fast mode), conversion 5 µs,
pas de registre de configuration : un seul byte de commande par lecture.
Pin-compatible avec l'ADS7828 (12 bits) si besoin de monter en résolution.

## Interface

- **Type** : I²C esclave (standard 100 kHz / fast 400 kHz / high-speed 3,4 MHz)
- **Adresse I²C (7 bits)** : `10010·A1·A0` → **0x48** (A1=A0=GND) à **0x4B** (A1=A0=VDD), 4 devices max par bus
- **Protocole** : write 1 byte de commande → read 1 byte de donnée (D7..D0). Aucun registre.

## Byte de commande

| Bit | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|---|---|---|---|---|---|---|---|---|
| Nom | SD | C2 | C1 | C0 | PD1 | PD0 | X | X |

- **SD** : 1 = single-ended (vs COM), 0 = différentiel
- **C2-C0** (single-ended, mapping non séquentiel !) : `000`=CH0, `001`=CH2, `010`=CH4, `011`=CH6, `100`=CH1, `101`=CH3, `110`=CH5, `111`=CH7
- **PD1-PD0** : `00` power-down entre conversions, `01` **réf. interne OFF + ADC ON** (mode réf. externe/ratiométrique), `10` réf. ON + ADC OFF, `11` réf. ON + ADC ON

Exemple : lire CH3 en single-ended, réf. externe → commande `0b1101_0100` = `0xD4`.

## Séquence d'initialisation

```
1. Aucune init requise — pas de registre de configuration, pas de WHO_AM_I.
2. Probe : write du byte de commande (ex. 0x84 = CH0, SD=1, PD=00) → ACK attendu.
3. Chaque lecture : write commande (1 byte) puis read résultat (1 byte).
4. Réf. externe sur VDD : utiliser PD1PD0=01 pour couper la réf. interne
   (sinon elle drive REFIN via 110 Ω et fausse tout).
```

## Exemple ESP-IDF

```c
#include "driver/i2c_master.h"

#define ADS7830_ADDR        0x48  // A1=A0=GND
// SD=1 single-ended, PD=01 (ref interne OFF, ratiometrique sur VDD)
static const uint8_t ads7830_cmd[8] = {
    0x84, 0xC4, 0x94, 0xD4, 0xA4, 0xE4, 0xB4, 0xF4  // CH0..CH7 dans l'ordre
};

static esp_err_t ads7830_read(i2c_master_dev_handle_t dev, uint8_t ch, uint8_t *out)
{
    if (ch > 7) return ESP_ERR_INVALID_ARG;
    return i2c_master_transmit_receive(dev, &ads7830_cmd[ch], 1, out, 1, 100);
}
```

## Notes spécifiques projet

- **EscapeBox face avant** : U_ADC lit les 8 contrôles (SL1-SL4 faders + RV1-RV4 pots
  rotatifs 10 kΩ). Remplace les 3 entrées ADC ESP32 (GPIO1-3) → libère GPIO1/2 pour
  les toggles SW1/SW2 et GPIO3 en réserve.
- **Câblage** : REFIN → 3V3 (ratiométrique, PD=01), COM → GND, A1/A0 → GND (adresse 0x48),
  100 nF de découplage VDD + 1 µF sur REFIN recommandés.
- **Adresse 0x48 (7 bits)** : libre sur notre bus. Ne pas confondre avec l'ancienne
  notation 8 bits « 0x48 » du PN532 (qui est 0x24 en 7 bits) — pas de collision réelle.
- Charge du sampling cap 25 pF via le curseur 10 kΩ : τ ≈ 0,25 µs, négligeable au
  rythme de polling (~50 Hz par voie).
