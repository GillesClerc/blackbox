# bq24075

> _Synthèse générée par component-research. Datasheet PDF : [./bq24075.pdf](./bq24075.pdf)_

**Fabricant** : Texas Instruments
**Catégorie** : Chargeur LiPo linéaire 1 cellule + power path management (DPPM)
**Référence officielle** : BQ24075RGTR (datasheet commun BQ24072/73/74/75/79, SLUS810N rév. octobre 2021)
**Datasheet source** : https://www.ti.com/lit/ds/symlink/bq24075.pdf

## Vue d'ensemble

Chargeur Li-Ion/LiPo linéaire 1 cellule (4,2 V) jusqu'à 1,5 A avec power path dynamique
(DPPM) : alimente le système sur OUT tout en chargeant la batterie indépendamment, et bascule
en « battery supplement mode » si la source ne suffit pas. Compatible USB (limites 100 mA /
500 mA / programmable via ILIM), VIN-DPM contre les sources USB faibles, protection
surtension entrée 28 V max (suspension > 6,6 V). Le BQ24075 ajoute la pin SYSOFF
(déconnexion batterie) ; OUT régulé à 5,5 V typ quand l'entrée est présente.
Configuration 100 % matérielle par résistances — aucun bus de contrôle.

## Package & Footprint

- **Package** : VQFN-16 (RGT) avec pad thermique exposé
- **Dimensions** : 3 × 3 mm
- **Pitch** : 0,5 mm
- **Empreinte LCSC** : C15464 (BQ24075RGTR)
- **Pad thermique** : à connecter au plan GND (même potentiel que VSS), vias thermiques — mais VSS pin 8 reste la masse principale du circuit

## Pinout (BQ24075, VQFN-16 RGT)

| Pin | Nom | Direction | Fonction |
|---|---|---|---|
| 1 | TS | IN | Entrée thermistance NTC 10k du pack batterie. **Sans thermistance : une résistance fixe 10 kΩ TS→VSS** (maintient une tension valide, sinon charge suspendue) |
| 2, 3 | BAT | I/O | Sortie étage de charge + sense batterie. Vers borne + LiPo. Bypass 4,7–47 µF vers VSS |
| 4 | CE | IN | Charge enable **actif bas**. LOW = charge ON. Pull-down interne ~285 kΩ, ne pas laisser flottant |
| 5 | EN2 | IN | Sélection limite courant d'entrée (voir table EN1/EN2). Pull-down interne ~285 kΩ |
| 6 | EN1 | IN | idem EN2 |
| 7 | PGOOD | OUT | Open-drain, LOW = source d'entrée valide. Pull-up 1 k–100 kΩ ou LED |
| 8 | VSS | PWR | Masse. Toujours connectée (le pad thermique ne remplace pas cette pin) |
| 9 | CHG | OUT | Open-drain, LOW = charge en cours. High-Z = charge finie ou désactivée. Pull-up 1 k–100 kΩ ou LED |
| 10, 11 | OUT | OUT | Sortie système (power path). 5,5 V typ sur entrée valide, sinon VBAT (sauf SYSOFF haut). Bypass 4,7–47 µF |
| 12 | ILIM | IN | Programmation limite courant d'entrée (mode EN2=H/EN1=L) : R 1,1 k–8 kΩ vers VSS. **ILIM flottant = toute charge désactivée** |
| 13 | IN | IN | Entrée DC 4,35–6,4 V (suspension au-dessus, destruction > 28 V). Bypass 1–10 µF vers VSS |
| 14 | TMR | IN | Timers de sécurité : flottant = défauts (précharge 30 min, charge 5 h), R 18–72 kΩ = programmable, **GND = timers désactivés** |
| 15 | SYSOFF | IN | HIGH = déconnecte la batterie de OUT. **GND pour fonctionnement normal, ne pas laisser flottant** |
| 16 | ISET | I/O | Programmation courant de charge rapide : R 590 Ω–8,9 kΩ (1 %) vers VSS. La tension sur ISET reflète le courant de charge réel (monitoring). Flottant = charge désactivée |

## Interface

- **Type** : configuration matérielle pure (résistances + niveaux logiques). Aucun bus I²C/SPI.
- **Statuts vers MCU (optionnel)** : PGOOD et CHG en open-drain → pull-up vers 3V3 et lecture GPIO.
- **Monitoring courant de charge (optionnel)** : tension ISET ∝ I_CHG réel → lisible sur un ADC.

## Dimensionnement (formules)

```
R_ISET = K_ISET / I_CHG        → 890 AΩ / 1,0 A  = 890 Ω   (charge 1 A)
R_ILIM = K_ILIM / I_IN_MAX     → 1610 AΩ / 1,5 kΩ ≈ 1,07 A  (mode EN2=H/EN1=L)
R_TMR  = t_MAXCHG / (10 × 48 s/kΩ)   — ou TMR→GND pour désactiver les timers
TS sans NTC : R fixe 10 kΩ TS→VSS (§9.3.6 et table 7-1 du datasheet)
```

## Séquence d'initialisation

```
1. Aucune — démarrage autonome dès VIN > UVLO et VIN > VBAT + 80 mV.
2. Au power-on : test court-circuit ILIM/ISET, détection batterie, puis précharge
   (VBAT < 3,0 V) ou charge rapide. Inrush limité par séquence propriétaire (USB-IF).
3. Côté firmware : rien d'obligatoire. Option : lire PGOOD/CHG sur GPIO pour
   afficher l'état de charge.
```

## Exemple ESP-IDF

Pas de driver nécessaire. Lecture optionnelle des statuts open-drain :

```c
// PGOOD / CHG : open-drain, pull-up externe 100k vers 3V3_D.
// LOW = actif (source valide / charge en cours).
static void charger_status_read(gpio_num_t pgood, gpio_num_t chg)
{
    bool input_ok  = (gpio_get_level(pgood) == 0);
    bool charging  = (gpio_get_level(chg) == 0);
    ESP_LOGI("charger", "input=%s charge=%s",
             input_ok ? "OK" : "absent", charging ? "en cours" : "terminee/off");
}
```

_(dans l'EscapeBox les pins PGOOD/CHG ne sont pas câblées au MCU — budget GPIO épuisé ; pull-ups optionnels prévus pour debug sonde)_

## Notes spécifiques projet

- **EscapeBox** : U8 sur Main board. R_ISET = 890 Ω → I_CHG = 1,0 A. R_ILIM = 1,5 kΩ.
- CE→GND (charge ON), SYSOFF→GND (obligatoire, jamais flottant), TMR→GND (timers off).
- **TS : une seule résistance 10 kΩ TS→VSS** (pas de thermistance dans le pack LiPo).
  ⚠ L'ancien schéma utilisait un diviseur 10k/10k vers VSYS — incorrect : TS est polarisée
  par une source de courant interne 75 µA, un diviseur vers VSYS fausse la tension et peut
  suspendre la charge silencieusement (CHG reste bas → aucun symptôme visible).
- **EN1/EN2 : mode RILIM retenu** (décision 2026-06-11) : EN2=VSYS (H), EN1=GND (L)
  → entrée limitée à ~1,07 A par R_ILIM = 1,5 kΩ, la charge atteint son 1 A.
  Implique un chargeur USB ≥ 1,5 A (pas un vieux port USB-A 500 mA — VIN-DPM
  n'est actif qu'en modes USB100/500, EN2=L).
- OUT = VSYS (5,5 V sur USB, VBAT sur batterie) → alimente les 2 AP2112K + MT3608.
