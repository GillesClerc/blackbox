  ESCAPEBOX — SCHEMA DETAILLE : ECRANS (YEUX + BOUCHE)
  2x GC9A01 1.3" ronds (SPI3) + 1x SSD1680 e-ink 2.9" (SPI2)
══════════════════════════════════════════════════════════════════════════════


  ┌─────────────────────────────────────────────────────────────────────────┐
  │        YEUX — 2x GC9A01 1.3" rond 240×240 (SPI3 partage)            │
  └─────────────────────────────────────────────────────────────────────────┘

  Les deux ecrans GC9A01 partagent MOSI/SCLK/DC/RST sur le bus SPI3.
  Seul le CS differencie l'oeil gauche (GPIO40) de l'oeil droit (GPIO14).

  SPI mode 0, max 80 MHz.
  Pas de MISO (ecriture seulement).
  Animations yeux a ~15-25 FPS par ecran en rafraichissement alterne.

  Note CS : GPIO40 = CS oeil gauche (SPI3 natif).
  GPIO14 = CS oeil droit (ex-CS XPT2046, recycle).


  ┌─────────────────────────────────────────────────────────────────────────┐
  │              TIMING SPI3 — Partage 2x GC9A01                         │
  └─────────────────────────────────────────────────────────────────────────┘

  Le driver ESP-IDF SPI gere l'arbitrage via les CS :

  CS_L (GPIO40) ──┐
                   ├── Jamais actifs en meme temps
  CS_R (GPIO14) ──┘

  - Rafraichissement oeil gauche : CS_L low, SPI @ 80 MHz, burst DMA
  - Rafraichissement oeil droit : CS_R low, SPI @ 80 MHz, burst DMA
  - Strategie : double buffer par oeil, alternance a chaque frame

  Bande passante : 240×240×2 bytes × 80 MHz SPI ≈ 27 FPS par oeil
  En alternance : ~13-14 FPS simultane (suffisant pour animations yeux)
  Optimisation : ne rafraichir que la zone modifiee (pupille, paupiere)
  reduit le debit a ~30-50% → retour a ~25 FPS effectif


  ┌─────────────────────────────────────────────────────────────────────────┐
  │       BOUCHE — SSD1680 e-ink 2.9" 296×128 (SPI2)                    │
  └─────────────────────────────────────────────────────────────────────────┘

  Ecran e-ink utilise comme "bouche" du personnage, sous les deux yeux.
  Affiche le texte mot-a-mot (style Animal Crossing) via partial refresh.

          ESP32-S3                     SSD1680 module (Waveshare 2.9")
       ┌──────────┐                  ┌──────────────────────┐
       │          │                  │                      │
       │ GPIO11 ──┼── MOSI ────────→│ DIN (data in)        │
       │          │                  │                      │
       │ GPIO12 ──┼── SCLK ────────→│ CLK (clock)          │
       │          │                  │                      │
       │ GPIO10 ──┼── CS ──────────→│ CS (active low)      │
       │          │                  │                      │
       │ GPIO9 ───┼── DC ──────────→│ DC (data/command)    │
       │          │                  │                      │
       │ GPIO8 ───┼── RST ─────────→│ RST (active low)     │
       │          │                  │                      │
       │ GPIO13 ──┼── BUSY ←────────│ BUSY (active low)    │
       │          │                  │                      │
       │          │                  │ VCC ─── 3.3V_D       │
       │          │                  │ GND ─── GND           │
       └──────────┘                  └──────────────────────┘

  SPI mode 0, max 20 MHz (e-ink plus lent que TFT).
  BUSY : le SSD1680 tire cette pin LOW pendant un refresh.
  Le firmware attend BUSY HIGH avant d'envoyer la commande suivante.

  Modes de rafraichissement :
  - Full refresh : ~3 secondes (noir → blanc → image). A utiliser
    entre les phases pour eliminer le ghosting.
  - Partial refresh : ~0.3 secondes. Pour affichage texte mot-a-mot
    (~3 mots/seconde, synchronise avec les syllabes audio AC).
  - Apres ~20 partial refresh consecutifs, faire un full refresh
    pour nettoyer les artefacts residuels.

  Orientation : paysage (296 pixels de large × 128 pixels de haut).
  Zone active : 67 × 29 mm — forme rectangulaire horizontale ("bouche").
  Driver Espressif : esp_lcd_ssd1681 (compatible SSD1680).
