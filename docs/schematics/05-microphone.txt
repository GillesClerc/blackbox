  Le ICS-43434 est un micro bottom-port : le son entre par
  le dessous du composant. Il faut :
  1. Un trou Ø1mm dans le PCB sous le micro
  2. Un trou Ø2-3mm dans le boitier aligne avec le trou PCB
  3. Pas de piste de cuivre sous le trou acoustique
  4. Zone de silence (pas de bruit digital) autour du micro

  Distance micro → ESP32 : garder les traces I2S courtes (< 5cm).
  Le micro est sur le PCB main, pas sur un satellite.


  ┌─────────────────────────────────────────────────────────────────────────┐
  │               SPECIFICATIONS AUDIO                                   │
  └─────────────────────────────────────────────────────────────────────────┘

  - Sensibilite : -26 dBFS (typ.)
  - SNR : 65 dBA
  - Bande passante : 50 Hz - 20 kHz (avec filtre HP interne)
  - Resolution : 24-bit I2S
  - Consommation : ~1.4 mA actif, ~6 µA sleep
  - Frequence clock : 1.024 - 4.096 MHz (typ. 2.822 MHz @ 44.1 kHz)

  Utilisation firmware :
  - Detection niveau sonore (peak level) pour interactions
  - Detection souffle (basse frequence, < 500 Hz)
  - Reconnaissance commande vocale (future, optionnel)
  - Le micro ne tourne PAS en permanence — active a la demande
    par le scenario pour economiser energie
